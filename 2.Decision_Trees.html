Es un método de ensamblaje que utiliza múltiples árboles de decisión para mejorar la robustez y reducir el riesgo de sobreajuste. Es bueno para manejar una gran cantidad de características y es generalmente resistente a los valores atípicos.

Para realizar un análisis detallado de los resultados obtenidos por los modelos que desarrollemos, utilizaremos las siguientes métricas:

- Tiempo de entrenamiento 
- MSE (Mean Squared Error)
- RMSE (Root Mean Squared Error)
- MAE (Mean Absolute Error)
- Complementario de R² (1 - R²).

También se tendrán en cuenta las características específicas del conjunto de datos para entender mejor el rendimiento de cada modelo.

def evaluacion(model, X_test, y_test):
    metricas = {}
    # Predecimos los valores de 'Precio' para el conjunto de prueba
    y_pred = model.predict(X_test)
    
    # MSE - Error Cuadrático Medio
    metricas['MSE'] = mean_squared_error(y_test, y_pred)

    # RMSE - Raíz del Error Cuadrático Medio
    metricas['RMSE'] = np.sqrt(metricas['MSE'])

    # MAE - Error Absoluto Medio
    metricas['MAE'] = mean_absolute_error(y_test, y_pred)

    # MedAE - Mediana del Error Absoluto
    metricas['MedAE'] = median_absolute_error(y_test, y_pred)

    # Coeficiente de Determinación, R²
    metricas['R2'] = r2_score(y_test, y_pred)

    # Imprimir resultados
    print("MSE:", metricas['MSE'])
    print("RMSE:", metricas['RMSE'])
    print("MAE:", metricas['MAE'])
    print("MedAE:", metricas['MedAE'])
    print("R²:", metricas['R2'])
    
    return metricas

def comparar_metricas(modelos_metricas, nombres_modelos):
    metricas = ['MSE', 'RMSE', 'MAE', 'MedAE', 'R2', 'MAPE', 'Tiempo de entrenamiento']
    
    data = {metrica: [modelo[metrica] for modelo in modelos_metricas] for metrica in metricas}
    df = pd.DataFrame(data, index=nombres_modelos)

    # Iteramos sobre cada métrica y crear una gráfica de barras
    for metrica in metricas:
        # Obtenemos los valores para la métrica actual de cada modelo
        valores = [modelo[metrica] for modelo in modelos_metricas]

        # Creamos la gráfica de barras
        plt.figure(figsize=(10, 6))
        plt.bar(nombres_modelos, valores, color=['blue', 'green', 'red', 'purple'])
        plt.title(f'Comparación de {metrica}')
        plt.ylabel(metrica)
        plt.xlabel('Modelos')
        for i, v in enumerate(valores):
            plt.text(i, v + max(valores)*0.01, f"{v:.2f}", ha='center')
        plt.show()
        
    return df

Análisis Comparativo:¶

Como podemos observar en esta primera aproximación creada utilizando cross – validation para determinar cuál es la arquitectura de árbol de regresión que mejor se ajusta a nuestros datos. Si tuviéramos que escoger uno solo para realizar la búsqueda de hiperparámetros con optimización bayesiana, el mejor sería Random Forest, aunque seguido muy de cerca por Light GBM, el cuál tarda muchísimo menos en entre nar. No obstante, al estar los resultados de todos los modelos muy parejos, vamos a aplicar optimización bayesiana sobre todos ellos, combinándolos de nuevo con validación cruzada para mitigar así el sobreajuste. Tras realizar todo el proceso de optimización, se ha dividido nuestro conjunto en datos de entrenamiento y test en una proporción de 0.8 y 0.2 respectivamente utilizando el método model_selection.train_test_split de sklearn y mezclando los datos de manera aleatoria. def sobreajuste(model, X_train=X_train, X_test=X_test, y_train=y_train, y_test=y_test):
    y_pred = model.predict(X_test)
    
    # Obtenemos el error en cada iteración
    test_score = [mean_squared_error(y_test, y_pred) for y_pred in model.staged_predict(X_test)]
    train_score = [mean_squared_error(y_train, y_pred) for y_pred in model.staged_predict(X_train)]
    
    # Creamos gráfico
    plt.figure(figsize=(12, 6))
    plt.plot(train_score, label='Training Error')
    plt.plot(test_score, label='Validation Error')
    plt.title('Error de Entrenamiento vs. Error de Validación')
    plt.xlabel('Número de Árboles')
    plt.ylabel('Error Cuadrático Medio')
    plt.legend()
    plt.show()

def objective_rf(space, X_train, y_train, X_val, y_val):
    model = RandomForestRegressor(**space)
    # Realizamos validación cruzada y calcular el error medio
    kf = KFold(n_splits=5, shuffle=True, random_state=42)
    mae_scores = cross_val_score(model, X, y, cv=kf, scoring='neg_mean_absolute_error')
    mean_mae = -np.mean(mae_scores)  # Convertimos a positivo
    return {'loss': mean_mae, 'status': STATUS_OK}

def objective_gb(space, X_train, y_train, X_val, y_val):
    model = GradientBoostingRegressor(**space)
    # Realizamos validación cruzada y calcular el error medio
    kf = KFold(n_splits=5, shuffle=True, random_state=42)
    mae_scores = cross_val_score(model, X, y, cv=kf, scoring='neg_mean_absolute_error')
    mean_mae = -np.mean(mae_scores)  # Convertimos a positivo
    return {'loss': mean_mae, 'status': STATUS_OK}

def objective_xgb(space, X_train, y_train, X_val, y_val):

Estos errores se verán aún más incrementados al usar un error cuadrático co mo MSE. Si nos fijamos en la mediana de los errores absolutos podemos ver que esta se sitúa entre 6000€ y 13000€ dependiendo del modelo.
